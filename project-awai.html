Kotlin MVVM Jetpack Navigation ViewBinding
Time

Apr 2026 – Present

Kotlin, MVVM, Jetpack Navigation, ViewBinding

In progress

Project Overview

A client-facing Android prototype designed to support Tamariki, Rangatahi, parents, and caregivers through quick response and information features.

Key Features / Responsibilities

  • Implemented core screens including Home, Quick Contacts, Contact Management, Profile, Information pages, Help Me Decide, quiz/result pages, and alert simulation flows.
  • Built a single-Activity, multi-Fragment MVVM structure using Jetpack Navigation and ViewBinding to improve navigation consistency and maintainability.
  • Participated in the first online client meeting, demonstrated the prototype, collected feedback, and refined the app using User-Centred Design principles.
  • Designed a mobile-first interface with quick response flows, English/Māori language considerations, simulated contact data, and AWAI-inspired visual styling.

Technical Implementation

The prototype uses a single Android activity with multiple fragments connected through Jetpack Navigation. ViewBinding keeps UI code clearer, while the MVVM structure separates screens, navigation flows, and simulated data handling for easier iteration.

What I Learned

Practical Android prototyping, client-facing demo preparation, navigation structure, ViewBinding usage, mobile-first UI thinking, and user-centred iteration.
